Arquitectura interna

Resumen de los componentes de la src actual de Itsuki

Pipeline

  1. Lexer en lexer.c: tokeniza (keywords, operadores, macros).
  2. Parser en parser.c: crea nodos AST (NodoAST).
  3. Evaluador/Compiler: puede evaluar AST y tambien compilar a bytecode.
  4. VM en vm.c: ejecuta opcodes sobre stack y closures.

AST y tipos

  • Tipos AST: AST_FUNCION, AST_CLASE, AST_ENUM, AST_TRY_CATCH, AST_IMPORTAR, AST_C_EXTERN, etc.
  • Tokens clave: TOKEN_SEA, TOKEN_FUNCION, TOKEN_CLASE, TOKEN_ENUM, TOKEN_INTENTAR, TOKEN_LANZAR.
  • Sistema de tipos con TipoDato y TipoExacto.

VM y bytecode

La VM define opcodes como OP_CONSTANTE, OP_LLAMAR, OP_RETORNAR, OP_TRY_BEGIN, OP_LANZAR, OP_IMPORTAR.

La estructura Chunk guarda codigo y constantes para ejecutar o serializar en .sukiby.

Memoria y GC

  • Gestion de objetos con gc_alloc, gc_marcar y gc_recolectar.
  • Modo manual opcional desde CLI con --no-gc.
  • Builtin liberar() disponible para liberar explicitamente.

FFI y modulos

  • Interop C con c_incluir y c_extern.
  • Bundling y glue code para externos durante -o y -c.
  • Sistema de modulos basado en imports y carpeta modules/.

Tooling adicional

  • lsp.c: servidor de lenguaje.
  • formatter.c: comando itsuki fmt.
  • linter.c: comando itsuki lint.
  • tools/: gestor de paquetes itsuki-cy.
