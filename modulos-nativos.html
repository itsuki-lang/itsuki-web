Modulos Nativos C

Itsuki v5.0 permite cargar modulos nativos compilados como librerias compartidas para alto rendimiento.

Concepto

  • Escribes el modulo en C.
  • Exportas una funcion obligatoria: itsuki_module_init.
  • Compilas a .dll (Windows) o .so (Linux).
  • Importas en Itsuki con importar "modules/mi_modulo" como mm.

Plantilla minima

#include "itsuki_ext.h"

static const ItsukiApi* g_api = NULL;

static Result fn_doble(Result args[], int n_args) {
    if (n_args < 1 || args[0].tipo != TIPO_NUMERO) {
        g_api->raise(ERROR_ARGUMENTO, "doble(x) requiere numero");
        return g_api->make_null();
    }
    return g_api->make_number(args[0].n * 2.0);
}

ITSUKI_EXT_EXPORT int ITSUKI_EXT_CALL itsuki_module_init(const ItsukiApi* api, ItsukiModule* module) {
    if (!api || !module) return 0;
    if (api->version != ITSUKI_EXT_API_VERSION) return 0;
    g_api = api;

    if (!api->export_const(module, "VERSION", api->make_string("1.0.0"))) return 0;
    if (!api->export_native(module, "doble", fn_doble)) return 0;
    return 1;
}

Compilar en Windows

gcc -shared -O3 -Wall -I./src/headers modules/mi_modulo.c -o modules/mi_modulo.dll -lm

Compilar en Linux

gcc -shared -fPIC -O3 -Wall -I./src/headers modules/mi_modulo.c -o modules/mi_modulo.so -lm

Compilar con Makefile de modules/

cd modules
make        # compila todos los modulos .c
make math   # compila solo math.c
make clean

Uso desde Itsuki

importar "modules/math" como math

print(math.PI)
print(math.factorial(5))
print(math.mcd(84, 30))

Errores comunes

  • No se pudo abrir el modulo: revisa nombre, ruta y que el binario exista.
  • Falta itsuki_module_init: la libreria no exporta la entrada esperada.
  • El ejecutable de Itsuki no carga nativos: recompila el core con el runtime de extensiones habilitado.
