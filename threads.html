Biblioteca threads

Modulo nativo en C implementado en modules/threads.c. Expone creacion de hilos, sincronizacion y utilidades de concurrencia.

Importacion

importar "modules/threads" como threads

Constantes de estado

Constante Descripcion
threads.STATUS_NEW Thread creado, aun no corriendo.
threads.STATUS_RUNNING Thread en ejecucion.
threads.STATUS_DONE Thread finalizado correctamente.
threads.STATUS_FAILED Thread termino con error.
threads.STATUS_CANCELLED Thread cancelado.

API base y tareas

Funcion Firma Descripcion
threads.version () Version del modulo threads.
threads.hardware_threads () Cantidad de hilos de hardware visibles.
threads.now_ms () Tiempo actual en ms.
threads.sleep_ms (ms) Pausa en milisegundos.
threads.yield_now () Cede CPU al scheduler.
threads.spawn_sleep (ms) Crea thread que duerme.
threads.spawn_system (cmd) Ejecuta comando del sistema en thread.
threads.spawn_process (cmd) Alias de spawn_system.
threads.spawn_itsuki (script, [args], [exe_path]) Ejecuta otro script Itsuki en segundo plano.
threads.spawn_sum (inicio, fin, [paso]) Suma rango en un thread.
threads.spawn_fibonacci (n) Calcula fibonacci en un thread.
threads.spawn_udp_sender ([socket], host, port, payload, count, [delay_ms]) Envia UDP en background.

Ciclo de vida de threads

Funcion Firma Descripcion
threads.join (id) Espera fin y retorna mapa de resultado.
threads.join_timeout (id, timeout_ms) Espera con timeout; retorna mapa con done.
threads.try_join (id) Join no bloqueante.
threads.detach (id) Desacopla thread para no hacer join.
threads.cancel (id) Solicita cancelacion.
threads.status (id) Retorna estado numerico.
threads.is_running (id) Retorna true si sigue corriendo.
threads.count_threads ([running_only]) Cuenta threads registrados.
threads.list_threads ([running_only]) Lista info de threads en array.
threads.destroy (id) Libera registro de thread finalizado.
threads.cleanup () Limpia entradas detached finalizadas.

Sincronizacion

El modulo tambien exporta APIs para mutex, rwlock, condvar, semaforos, barreras y atomics:

  • mutex_new / mutex_lock / mutex_try_lock / mutex_unlock / mutex_destroy
  • rwlock_new / rwlock_rdlock / rwlock_wrlock / rwlock_try_rdlock / rwlock_try_wrlock / rwlock_rdunlock / rwlock_wrunlock / rwlock_destroy
  • cond_new / cond_wait / cond_signal / cond_broadcast / cond_destroy
  • sem_new / sem_wait / sem_try_wait / sem_post / sem_value / sem_destroy
  • barrier_new / barrier_wait / barrier_destroy
  • atomic_new / atomic_get / atomic_set / atomic_inc / atomic_dec / atomic_add / atomic_sub / atomic_cas / atomic_destroy

Ejemplo minimo

importar "modules/threads" como threads

sea tid = threads.spawn_sleep(200)
sea info = threads.join(tid)

print("thread=" + string(info["id"]))
print("status=" + string(info["status"]))
print("task=" + string(info["task"]))
